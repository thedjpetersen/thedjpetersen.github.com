Comparing web chat clients

June 04, 2012

I have been working the last two month on Subway an IRC web client. During this time I have discovered some of the other options that exist. I thought I would go ahead and share them with you.

Balloons.io

Balloons.io is a new webchat client built on node.js, socket.io and redis. This is probably my favorite out of the bunch due to the low amount of dependencies and the attractive appearance.

Kandan

Kandan is a new webchat client built on Ruby on Rails and event machine. The thing that strikes you the most about this client is the interface. It is extremely polished. The designer wrote a book based on his work designing the client. It took me 5 minutes to get this set up and running, I didn't have others to help me test it more extensively though.

Alice

Alice is probably the most mature option in the bunch. It runs as a server in the backend which handles the IRC connections and logs messages so you can see them when you log back in. It persists your connection to the server as well.

Holla

Holla is another open source application written in Ruby. It serves as both the server and the client. The logging is done in Redis, a data store system optimized for speed.

Candy

Candy is another client server option written in fully in javascript built on top of XMPP for communities to chat with each other. I didn't see whether or not logging/search was built in.

QWebIRC

QWebIRC is a online IRC client that has been around for more than 4 years. It was designed for use in on the quake network.
